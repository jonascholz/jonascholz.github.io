## Value Encoding

A spiking neuron communicates in discrete spikes. In the ideal case, it represents values by the exact timing of it's spike. For example, firing at time t=35 may represent value 35 and firing at time t=78 represents value 78.

<div id="spike-timing-diagram" style="width: 100%; margin: 1.5em auto;"></div>
<figcaption><strong>Figure 1:</strong> A neuron encodes the value 78 by staying silent until t=78, then firing once.</figcaption>
</figure>

If there are 100 timesteps in total, the neuron can represent 100 different values and only needs to send a single bit. This is assuming that silent timesteps have neglible compute cost, which is a big assumption, but maybe possible on neuromorphic hardware.

In exchange the neuron has to have a state, which is already at least 16-bit. Also we should note that a 6-bit number can already represent 127 bits, so it seems like the ANN does it cheaper. 

### The channel factor
You can add additional neurons to represent a value, but you are paying the same penalty for neuron states. That won't bring any gains. 

You can add more input connections into a neuron and that would actually help.
